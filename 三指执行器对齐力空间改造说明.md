# 三指执行器对齐力空间改造说明

## 目标叙事

本次改造将策略学习目标从旧版的 scalar grasp force regression 转为 actuator-aligned per-finger force distribution learning。

旧版目标是预测一个虚拟三指合力：

```text
F_sum(t)
```

新版目标是预测三指夹爪可直接执行的手指力向量：

```text
F(t) = [F0(t), F1(t), F2(t)]
```

核心公式保持 reference-anchored residual adaptation 不变，只是从标量空间升级到三维执行器空间：

```text
F_pred_i(t) = F_ref_i(e) + g(t) * DeltaF_i(t)
```

向量形式：

```text
F_pred(t) = F_ref(e) + g(t) * DeltaF(t),    F in R^3
```

这样论文叙事仍然是“跨介质抓取中的参考锚定残差力自适应”，但输出空间与三指独立控制接口对齐。

## 标签定义

新增 finger-level 标签来自每根手指内部 4 个 z 轴传感器：

```text
F_i(t) = mean_j abs(z_i,j(t) - baseline_i,j)
```

其中：

- `i` 是 finger index，取值为 0, 1, 2
- `j` 是同一根手指上的 4 个 z sensor
- `baseline_i,j` 来自 contact 前窗口，默认 `pre_contact_median`
- 输出是非负的 finger normal contact intensity / normal pressure proxy

这个标签不再跨三根手指做 signed-sum，因此避免了三指圆盘 120 度分布下局部坐标轴不一致导致的虚假抵消。

```text

```

旧字段仍保留，避免已有实验和诊断脚本立刻失效。

### `src/cmg/models/modules.py`

新增 policy head：

```yaml
policy:
  head_type: state_residual_per_finger
```

该 head 使用共享参数的 finger-conditioned policy：

```text
shared task/state context
+ finger embedding
-> shared finger policy
-> [F0, F1, F2]
```

新增输出：

```text
finger_force_pred                 [B*W, 3]
finger_force_base                 [B*W, 3]
finger_force_interface_delta      [B*W, 3]
finger_force_interface_delta_raw  [B*W, 3]
```

同时保留旧 scalar 输出为三指均值，主要用于兼容旧日志入口：

```text
force_pred
force_base
force_interface_delta
```

### `src/cmg/models/system.py`

新增 per-finger reference override：

```text
finger_force_base = finger_reference_forces
finger_force_pred = finger_force_base + gate * finger_force_interface_delta
```

当 `use_reference_force_context: true` 时，residual context 使用每根手指自己的 `finger_reference_forces`。

### `src/cmg/losses.py`

新增 loss type：

```yaml
policy_loss:
  type: explicit_reference_delta_per_finger_smooth_l1
```

内部复用旧的 explicit reference-delta loss 结构，但将目标和 mask 切换到 per-finger 字段。

### `src/cmg/training.py`

新增 finger-space 指标：

```text

finger_stable_mae
finger_interface_mae
finger_control_overall_mae
finger_control_reference_mae
finger_control_interface_mae
finger_delta_interface_mae
finger0_control_interface_mae
finger1_control_interface_mae
finger2_control_interface_mae
```

并允许旧 checkpoint 初始化时忽略新增的 `policy_head.finger_*` 参数。

### `scripts/visualize_finger_force_labels.py`

新增三指标签诊断图。用于确认 `per_finger_z_abs_mean` 曲线是否合理。

## 当前实现边界

当前模型层第一版采用：

```text
global visual/context evidence
+ full tactile encoding
+ finger identity embedding
-> shared per-finger policy
```

也就是说，三根手指共享整体上下文和已有触觉编码，并通过 finger embedding 形成不同输出。更细粒度的“每根手指单独 tactile encoder / local tactile branch”可以作为下一版增强，不影响当前 actuator-aligned 输出叙事。

## 建议后续检查

训练后重点看：


## stage38b 调整

stage38 初跑后发现 best 停在 epoch 1，预测几乎等同于 reference force，说明 per-finger residual head 的学习信号偏弱。stage38b 在不改变数据口径和模型头结构的前提下做三个收敛性修正：

```text
configs/stages/stage38b_policy_fold1_per_finger_reference_delta_scale20_freeze.yaml
```

关键差异：

```yaml
freeze_modules:
  - attribute_head.object_projection

policy_loss:
  delta_normalization_scale: 20.0

selection_metric: finger_control_interface_mae
tie_breakers:
  - finger_delta_interface_mae
  - finger_control_overall_mae
  - finger0_control_interface_mae
  - finger1_control_interface_mae
  - finger2_control_interface_mae
```

其中 `delta_normalization_scale` 从 `80.0` 降到 `20.0`，用于增强每指 residual delta 的梯度；`attribute_head.object_projection` 加入冻结范围，用于保持 Stage2 对齐表征不被 policy loss 轻微改写；best 选择和 tie-breaker 全部采用 per-finger 指标，旧标量合力指标仅作为兼容日志保留，不作为论文主线指标。

## stage38c / stage38d 对照

stage38b 跑满 12 个 epoch 后仍在改善，但 predicted residual 幅度仍明显小于 target delta。为区分“训练还没跑够”和“delta scale 仍偏保守”，新增两个对照配置：

```text
configs/stages/stage38c_policy_fold1_per_finger_reference_delta_scale20_long.yaml
configs/stages/stage38d_policy_fold1_per_finger_reference_delta_scale10.yaml
```

stage38c 只延长训练长度：

```yaml
policy_loss:
  delta_normalization_scale: 20.0
train:
  epochs: 24
  early_stopping_patience: 6
```

stage38d 只进一步增强 residual delta 梯度：

```yaml
policy_loss:
  delta_normalization_scale: 10.0
train:
  epochs: 12
  early_stopping_patience: 4
```

两者均继续冻结 `attribute_head.object_projection`，并继续只用 `finger_*` 指标做 best 选择和 tie-breaker。建议优先跑 stage38d；若 `stable_leakage_mean/p95` 明显升高或验证 MAE 反弹，再回到 stage38c 判断 scale=20 是否只是训练不足。

## stage38e：final-pred interface 监督与 residual guard

stage38c 的可视化显示预测曲线已经稳定贴近 reference，但 interface 段 residual 幅值偏保守，且整体更像 `reference + small positive residual`。stage38e 不改模型结构与数据口径，只在 stage38c 的基础上加入方案 A：

```text
configs/stages/stage38e_policy_fold1_per_finger_reference_delta_pred_interface_guard.yaml
```

关键差异：

```yaml
policy_loss:
  interface_weight: 0.25
  residual_interface_weight: 1.5
  residual_stable_zero_weight: 1.0
  residual_non_interface_weight: 0.3
  delta_normalization_scale: 20.0
```

其中 `interface_weight` 直接监督最终的 per-finger `finger_force_pred` 与 `finger_control_force_targets`，用于把预测幅值从 reference 附近拉向 interface target；但这一路 SmoothL1 没有经过 `delta_normalization_scale` 缩放，因此先采用较保守的 `0.25`。同时提高 stable/reference 段与 quiet/non-interface 段的 residual zero penalty，避免新增 final-pred 监督后在非 interface 区域产生 residual 泄漏。

建议从 stage38c best 初始化：

```powershell
.\.venv310\Scripts\python.exe -B scripts\train.py --project-root . --stage configs/stages/stage38e_policy_fold1_per_finger_reference_delta_pred_interface_guard.yaml --init-from runs/stage38c_policy_fold1_per_finger_reference_delta_scale20_long/checkpoints/best.pt
```

## stage38f：per-finger large-delta residual 增强

stage38e 证明 `interface_weight=0.25` 能把预测从 reference 附近稍微拉向 target，但 residual 幅值仍明显欠预测，且 stable leakage 已有上升。stage38f 保留 stage38e 的 final-pred interface 监督与 stable/non-interface guard，只对 per-finger 大幅 delta 的 residual 误差增加额外权重：

```text
configs/stages/stage38f_policy_fold1_per_finger_reference_delta_large_delta_guard.yaml
```

关键差异：

```yaml
policy_loss:
  interface_weight: 0.25
  residual_interface_weight: 1.5
  residual_large_delta_weight: 2.0
  sign_balanced_large_delta: true
  large_delta_threshold: 100.0
  residual_large_delta_threshold: 8.0
  large_delta_weight_alpha: 1.0
  large_delta_weight_cap: 2.0
  residual_stable_zero_weight: 1.0
  residual_non_interface_weight: 0.3
```

其中 `residual_large_delta_threshold` 是 per-finger residual loss 的阈值，用于覆盖明显欠幅的手指窗口；`large_delta_threshold` 仍保留为 `100.0`，用于旧 scalar large-delta 诊断口径，避免日志里的旧指标与前序实验失去可比性。

建议从 stage38e best 初始化：

```powershell
.\.venv310\Scripts\python.exe -B scripts\train.py --project-root . --stage configs/stages/stage38f_policy_fold1_per_finger_reference_delta_large_delta_guard.yaml --init-from runs/stage38e_policy_fold1_per_finger_reference_delta_pred_interface_guard/checkpoints/best.pt
```

## stage38g / stage38h：先方向后幅值 curriculum

stage38f 将 per-finger predicted delta 幅值从 stage38e 的较低水平进一步抬高，但同时暴露出 wrong-sign rate 上升的问题。因此 stage38g / stage38h 采用两阶段 curriculum：先让模型学会 residual 方向，再恢复幅值增强。

新增 per-finger large-delta 诊断指标，阈值默认使用 `finger_large_delta_threshold`，若未配置则回退到 `residual_large_delta_threshold`：

```text
finger_large_delta_wrong_sign_rate
finger_large_delta_balanced_wrong_sign_rate
finger_large_delta_interface_mae
finger_large_delta_pred_abs_mean
finger_large_delta_target_abs_mean
```

stage38g 是 sign-first 阶段：

```text
configs/stages/stage38g_policy_fold1_per_finger_reference_delta_sign_first.yaml
```

关键差异：

```yaml
policy_loss:
  interface_weight: 0.15
  residual_interface_weight: 0.75
  residual_large_delta_weight: 0.5
  residual_large_delta_sign_weight: 2.0
  large_delta_sign_margin: 0.10
  residual_large_delta_threshold: 8.0

```

stage38h 是 magnitude follow-up 阶段：

```text

```

关键差异：

```yaml
  residual_large_delta_weight: 2.5

```

推荐先从 stage38f best 初始化 stage38g；stage38g 完成后，再用 stage38g best 初始化 stage38h。
