# 项目背景信息

更新日期：2026-04-28

## 1. 项目目标

本项目研究“跨介质视触抓取”。

目标是让机械臂从水下抓取物体，并在穿过水-气界面时，结合：

- 视觉
- 触觉
- 介质状态

输出合理的抓取控制力。

项目的理想控制形式，不再被理解为“直接从感知回归最终力值”，而更像是：

```text
最终控制力 = 基准力 + 界面修正量
```

更细一点可以写成：

```text
force_pred = base_force + interface_gate * interface_delta
```

当前最新证据则进一步表明，工程上更可行的形式是：

```text
force_pred = reference_force + gate * delta
```

后续再逐渐扩展成：

```text
force_pred = reference_force + base_correction + gate * delta
```

## 2. 项目主训练链路

当前项目按阶段组织：

1. `Stage1` 感知
2. `Stage2` 视触对齐
3. `Stage3 / 36` 策略头训练与诊断
4. `Stage4 / 4b` 联合训练或策略微调

这条链路已经在工程上打通。  
也就是说，项目已经不是设计阶段，而是 **离线 MVP 已完成，当前处于策略性能诊断和训练路径重构阶段**。

## 4. 当前工程状态判断

### 4.1 感知模块已经有效

从较新的 `Stage4b` 结果看：

- `medium_macro_f1` 已有较好水平
- `medium_f1_interface` 已明显高于早期阶段
- `Water / Air / Interface` 的识别能力已经是可用的

所以当前主矛盾不是“介质识别完全不行”，而是：

**策略控制力没有学好。**

### 4.2 Stage4b 暴露出的核心问题

`stage4b_joint_fold1_reference_delta` 的关键现象是：

- 感知指标不差
- 但 `control_interface_hit_rate_200` 仍然很低

这说明：

- 模型已经知道自己在什么介质里
- 但还不会在控制意义上给出合适力值

简而言之：

**感知是通的，控制还卡着。**

## 5. 已经确认的关键诊断结论

### 5.1 `delta` 分支本身不是不会学

OBJ012 tiny 实验已经说明：

- 当 `base` 不稳时，`delta` 容易塌在 0 附近
- 当 `base` 换成 oracle/reference 时，`delta` 立刻能学起来
- 再做 `delta` 归一化后，学习更稳

结论：

**`delta` 分支本身没有“能力性缺陷”。**

### 5.2 `base` 在 tiny 上能学，但对尺度很敏感

OBJ012 tiny 也说明：

- pure `base` 在小集上不是学不会
- 但必须处理好数值尺度

结论：

**`base` 问题里既有优化尺度问题，也有目标定义问题。**

### 5.3 `reference_force` 不是稳定对象常数，而是很强的样本级量

`fold1` 上的分组统计已经确认：

- `reference_force` 在对象内也会明显漂移
- 在相同条件组内也可能漂移很大

例如：

- `OBJ005 / clear + normal + normal`：约 `-806.8 -> -449.2`
- `OBJ010 / clear + normal + inverse`：约 `-395.7 -> -134.3`

结论：

**让模型纯猜一个 absolute `base`，天然就很难泛化。**

## 6. 最近两条 fold1 路线的结论

### 6.1 `fold1_base_norm`：pure learned absolute base 基本失败

对应配置：

`configs/stages/stage36b_policy_fold1_base_norm.yaml`

关键结果：

- `val_control_reference_mae ≈ 520.5`
- `val_control_interface_mae ≈ 519.3`
- `val_control_interface_hit_rate_200 ≈ 0.0577`

诊断里最关键的是：

- `reference_force_mean ≈ -539.2`
- `force_base_mean ≈ -4.8`
- `base_minus_reference_mean ≈ +534.4`

也就是说：

**模型几乎还停在 0 附近，没有真正学到 `reference_force` 的量级。**

结论：

- pure learned absolute base 仍然可以保留为 baseline
- 但它不应继续被视为最有希望的主线

### 6.2 `fold1_reference_anchor_delta_norm`：reference 直锚被强力验证

对应配置：

`configs/stages/stage36b_policy_fold1_reference_anchor_delta_norm.yaml`

核心做法：

- `base_source: reference_force`
- 冻结 `policy_head.base_*`
- 只训练 `delta`

和 `fold1_base_norm` 相比，控制指标出现数量级改善：

- `val_control_reference_mae: 520.5 -> 56.7`
- `val_control_interface_mae: 519.3 -> 68.9`
- `val_control_interface_hit_rate_200: 0.0577 -> 0.9375`
- `val_control_interface_bias: +513.9 -> -16.9`

诊断里最关键的是：

- `base_minus_reference_mean = 0.0`

这说明：

**把 measured `reference_force` 直接当锚点后，之前最大的 base 泛化问题被结构性绕开了。**

## 7. 当前最重要的判断

截至 2026-04-28，当前最重要的判断是：

1. `delta` 是能学的
2. pure learned absolute `base` 不是当前最值得推进的主线
3. measured `reference_force` 直锚在 fold1 上已经被强力验证
4. 当前主问题已经从“base 会不会学”收缩成：

**“在 reference 锚点已经给定的前提下，如何把 delta 学得更充分。”**

## 8. 当前推荐的项目优化方向

### 8.1 短期：保留 pure learned base 路线作为 baseline

保留原先三阶段路线，用来做正式对照：

1. `base-only normalized`
2. `learned base frozen + delta_norm`
3. `light joint fine-tune`

但它现在的定位只是：

- baseline
- 对照组

不是默认主线。

### 8.2 主线：转向 `reference_force` 直锚

当前更值得推进的主线是：

```text
force_pred = reference_force + gate * delta
```

随后再扩展到：

```text
force_pred = reference_force + base_correction + gate * delta
```

这样更像工程上靠谱的解法，因为：

- `reference_force` 已经提供了当前 episode 的锚点
- 不再需要模型盲猜 absolute base
- 模型只需要学习“相对当前锚点该补多少”

### 8.3 下一轮重点不是改 `base`，而是把 `delta` 学得更充分

当前直锚方案已经解决了最大的 `base` 泛化问题。  
所以 stronger 配置的重点应该是：

- 增强 interface residual 学习
- 降低过强的 stable zero 正则
- 提高 interface 采样强度
- 拉长训练

而不是继续回到 pure learned base 上硬拧。

### 8.4 数据治理优先继续看 reference 标签和可疑样本

当前依然建议优先检查：

- `reference_force_window_count`
- `reference_force_statistic`
- reference window 是否真的稳定
- 极端样本是否带偏 reference

优先审查样本可先看：

- val：`S0160`, `S0163`, `S0078`, `S0080`, `S0076`
- train：`S0016`, `S0037`, `S0039`, `S0044`

## 9. 接下来的最合适动作

当前最推荐的顺序是：

1. 把 `fold1_base_norm` 留作正式 baseline
2. 继续推进 `reference_anchor` 主线
3. 补并运行更强一些的 `fold1 reference_anchor_delta_stronger`
4. 持续用分支级诊断判断：
   - `base_minus_reference`
   - `delta_minus_target`
   - `force_base`
   - `force_interface_delta`
   - `control_interface_mae`
   - `control_interface_hit_rate_200`

## 10. 一句话总结

这个项目当前最值得记住的现实，不是“感知还差一点”，而是：

**感知已经够用了；真正决定项目能不能顺利推进的，是把策略主线从 pure learned absolute base 切换到 measured `reference_force` 直锚，再把 `delta` 学充分。**
