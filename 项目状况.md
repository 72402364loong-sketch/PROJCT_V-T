## 1. 项目目标与研究问题

本项目研究的是“跨介质视触抓取”问题。任务背景是：机械臂在水下抓取物体，随后夹持物体上提并穿过水-气界面，最终在出水后仍保持稳定抓取。

相较普通抓取，本任务的难点在于：

- 介质状态发生明显变化：Water、Interface、Air 三个阶段对抓取力需求不同。
- 视觉和触觉都重要：视觉提供对象外观和空间上下文，触觉提供接触状态、受力和介质变化信号。
- 控制目标不是静态常数：稳定水下段、界面穿越段和出水后稳定段的合理力输出不同。

项目最初尝试让模型直接从多模态输入回归最终抓取力。随着实验推进，目前主线已经收敛为更结构化的形式：

```text
force_pred = reference_force + gate * delta
```

其中：

- `reference_force`：当前 episode 内稳定抓取阶段测得的参考力，作为控制锚点。
- `gate`：界面修正是否生效以及生效强度，当前主要来自 medium/interface 概率。
- `delta`：相对参考力的界面修正量。

这个设计可以理解为一个 reference-anchored residual control model，而不是 from-scratch absolute force regressor。

```text
fragility_v2:
  fragile
  non_fragile        # robust + compliant

shape_profile_v2:
  open_vessel        # cup-like + bowl-like
  slender_or_narrow  # cylindrical + constricted-opening

surface_texture_v2:
  smooth
  textured_or_matte  # matte
```

当前结论：

- `shape_profile_v2` 是最适合作为主属性指标的标签。
- `fragility_v2` 在 stage1 可学，但在 stage2/shape-only 语境下不稳定，可能不是当前信号最可靠表达的属性。
- `surface_texture_v2` 因 val 缺类，不适合作为主指标。

### 3.3 运行限制与已遇到的问题

在 Windows 上使用 PyTorch DataLoader 多进程时曾出现：

```text

Error loading cublas64_12.dll or one of its dependencies.
```

当前规避方案：

```yaml

  num_workers: 0
  pin_memory: false
  persistent_workers: false
```

这会降低数据加载速度，但显著提高稳定性。

### 4.1 数据层

主要文件：

- `src/cmg/data/dataset.py`
- `src/cmg/data/tactile.py`
- `src/cmg/data/video.py`
- `src/cmg/data/splits.py`

主要功能：

- 读取 `samples.csv` 和 `windows.csv`。
- 按 split 过滤 sample。
- 按 window 构造视频帧和触觉片段。
- 默认过滤低有效率窗口：`valid_ratio_video < 0.8` 或 `valid_ratio_tactile < 0.8`。
- 触觉窗口重采样到固定长度 26 点。
- 触觉做 AC/DC 分解，高频和低频分别进入不同编码器。
- 支持 visual feature cache。
- 支持 `reference_force`、`delta_force_target`、`soft_gate_targets`、context window 等策略监督字段。
- 支持 `attribute_taxonomy: legacy / coarse_v2`。

### 4.2 视觉编码器

主要实现：

`src/cmg/models/modules.py::VisualEncoder`

默认配置：

```yaml
visual:
  backbone: open_clip_vit_b_16
  model_name: ViT-B-16
  pretrained: true
  pretrained_tag: openai
  freeze_backbone: true
  use_lora: true
  lora_rank: 8
  lora_alpha: 16
  lora_dropout: 0.0
  lora_target_semantics: [q_proj, v_proj]
  token_dim: 512
  proj_dim: 256
```

视觉流程：

```text
每个 window 多帧图像
-> OpenCLIP ViT-B/16 visual backbone
-> token_projection 到 512 维
-> VisualTemporalAttentionPooling 聚合窗口内帧
-> h_v
-> align_head 到 256 维并归一化，得到 z_v
```

LoRA 设计：

- 只注入 OpenCLIP transformer attention 的 Q/V 部分。
- 项目内自定义 `OpenClipQVLoraAttention`，对 fused `in_proj_weight` 的 Q/V 段加入低秩增量。
- 默认 `rank=8, alpha=16`。
- 轻量增强实验使用 `rank=16, alpha=32, dropout=0.05`。

stage1 与 stage2 的视觉差异：

- stage1 常使用 `visual_feature_cache_dir`，且 `use_lora: false`，不实时训练视觉 backbone。
- stage2 不使用 cache，开启 LoRA，实时跑 OpenCLIP + LoRA。
- stage2 常设置 `max_windows_per_encode: 2` 控制显存。

### 4.3 触觉编码器与高低频解耦

触觉输入维度：

```yaml
tactile:
  input_dim: 36
  num_taxels: 12
  axis_dim: 3
  content_encoder_type: cnn1d
  content_hidden_dim: 256
  content_layers: 2
  content_kernel_size: 3
  evidence_hidden_dim: 128
  evidence_kernel_size: 3
```

当前实现中触觉被拆成：

- `tactile_high`：高频/内容分支，用于 `TactileContentEncoder`。
- `tactile_low`：低频/证据分支，用于 `TactileEvidenceEncoder`。

高频内容分支：

```text
tactile_high
-> 1D CNN
-> TemporalAttentionPooling
-> z_content
-> align_head 得到 z_t
```

低频证据分支：

```text
tactile_low
-> 1D CNN
-> TemporalAttentionPooling
-> z_med_window
```

`z_med_window` 再进入 `MediumBeliefHead`，该模块是 GRU：

```text
z_med_sequence -> GRU -> medium logits/probs
```

这就是当前“高低频解耦”的代码实现基础：

- 高频内容分支服务对象属性、跨模态对齐和策略上下文。
- 低频证据分支服务介质阶段识别和 gate。

但需要说明：目前高低频解耦的理论支撑还不够强，因为尚缺少系统化的 high-only / low-only / shared-encoder ablation。

### 4.5 策略头

主要实现：

- `src/cmg/models/modules.py::PolicyHead`
- `src/cmg/models/system.py::CrossMediumSystem`

当前主策略形式：

```text
force_pred = reference_force + interface_gate * force_interface_delta
```

其中：

- `reference_force` 由 dataset 提供。
- `interface_gate` 当前主要使用 `medium_probs[:, 1]`，也测试过 soft/hard oracle gate。
- `force_interface_delta` 由 residual 分支预测。

已经实现过的策略扩展包括：

- `base_source: reference_force`
- `gate_source: medium_prob / soft_target / hard_phase`
- `PhaseAwareGateStabilizer`
- soft gate target 构造
- interface context embedding V1/V1b

### 6.4 oracle gate / context 实验

已测试：

- soft oracle gate
- hard oracle gate
- gated residual loss
- interface context V1/V1b

结论：

- oracle gate 可以使 stable leakage 接近 0，但没有超过 balanced 的 interface 主指标。
- hard oracle gate 比 soft oracle 略好，但仍不超过 balanced。
- simple additive context 没有带来净收益。
- 当前问题不是“gate 越干净越好”，medium_prob gate 的提前/滞后激活对 interface 控制可能有正贡献。

## 8. 当前主要问题

### 8.1 论文主线最强证据在 reference-anchored residual control

最强证据非常明确：

```text
pure learned absolute base -> control_interface_mae ≈ 519
reference_anchor_delta -> control_interface_mae ≈ 68
```

这足以支撑“reference anchor + residual delta”作为核心方法。

### 8.2 跨介质感知是可用但不是完美

medium/phase 识别整体可用：

- `medium_macro_f1` 常在 `0.84-0.87`。
- `medium_f1_interface` 常在 `0.65-0.72`。

但 interface 仍是难点，且当前优化应更关注：

- interface precision / recall
- onset / offset delay
- stable 段 false positive

而不是单纯 overall accuracy。

### 8.3 高低频解耦的直接证据还不够强

代码上已经存在高频内容分支和低频证据分支：

- 高频触觉服务属性、对齐、策略上下文。
- 低频触觉服务介质识别和 gate。

但目前缺少系统消融：

- high-only
- low-only
- shared tactile encoder
- visual-only
- tactile-only

因此论文中如果要强力主张“高低频解耦带来理论优势”，还需要补充实验。

### 8.4 属性理解目前只能支持 shape profile

当前比较稳妥的结论是：

- `shape_profile_v2` 是当前可支撑的主属性任务。
- `fragility` 和 `surface` 不适合作为主指标。

原因：

- fragility 更像材料/风险语义，当前视觉触觉信号未必足够可观测。
- surface 在 val 缺类，评估本身不完整。
- object-level unseen split 下对象数量少，容易出现 object identity bias。

### 8.5 数据规模限制明显

当前 fold1 val 只有 2 个 object，很多结论对 object 分布很敏感。模型变大或 LoRA rank 提高容易过拟合，不能简单用大模型/更深网络解决。

### 8.6 Windows 长跑环境存在资源问题

多进程 DataLoader 在当前 Windows + CUDA 环境下容易触发 pagefile / DLL 加载问题。当前靠 `num_workers=0` 稳定训练，但速度较慢。

## 11. 当前结论摘要

截至 2026-05-16，项目已经完成完整离线训练与诊断闭环，并形成如下阶段性判断：

1. 直接学习 absolute force/base 在 fold1 上泛化很差。
2. measured `reference_force` 作为 episode-level anchor 是当前最有效的结构改进。
3. `delta` 分支能学，但继续微调收益有限。
4. medium/interface 感知可用，但仍不是完美，interface 是主要难点。
5. 原始属性标签不适合直接支撑论文，需要 coarse taxonomy。
6. 当前属性主指标应收缩为 `shape_profile_v2`。
7. 高低频解耦已有代码实现，但仍需要针对性消融来支撑理论叙事。
8. 硬件上 GPU 充足，但 Windows 多进程 DataLoader/pagefile 是实际运行限制。
