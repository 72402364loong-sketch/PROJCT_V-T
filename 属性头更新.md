# 属性头更新

更新时间：2026-05-27

## 当前结论

Stage2 sample-pool 版本已经验证了一个关键信号：对象属性更适合 sample/object 级聚合评价，而不是只看 all-window 指标。

当前 best 结果来自：

```text
runs/stage2_alignment_fold1_lowmem_inv_attr_v2_sample_pool/checkpoints/best_metrics.json
```

关键指标：

```text
window-all attr macro F1                 0.4362
sample stable Water+Air mean-logits F1   0.5539
object stable Water+Air mean-logits F1   0.6667
attr_primary_source                      sample_model_pool
attr_primary_macro_f1                    0.5735
```

这说明“稳定接触窗口 + sample/object 聚合”的属性评价口径是有价值的。下一阶段不应急着把 metadata 输入 policy，而应先证明物理属性标签可靠、可学习、可解释。

## 目标

把 `data/annotations/object_physical_attributes.csv` 接入数据管线，建立物理属性头的第一版 sanity check：

```text
dry_mass_g: regression
capacity_ratio: masked regression
is_open_container: binary classification
water_capacity_g: optional masked regression
```

第一版物理属性头只作为：

```text
auxiliary supervision
mechanism diagnostic
representation probing
```

暂时不接入 policy / delta head / gate head。

## 第一阶段：数据管线

- Dataset 读取 `object_physical_attributes.csv`
- 通过 `object_id` join 到每个 sample/window
- 保留旧语义属性 `fragility / geometry / surface`
- 新增物理属性输出：
  - `dry_mass_g`
  - `dry_mass_g_normalized`
  - `is_open_container`
  - `full_water_mass_g`
  - `water_capacity_g`
  - `water_capacity_g_normalized`
  - `capacity_valid_mask`
  - `capacity_ratio`
  - `capacity_ratio_normalized`
  - `load_transition_class`
  - `shape_profile_ctrl`
  - `material_safety_class`
  - `surface_texture`

连续属性使用 train split 统计量归一化，保存到：

```text
data/processed/stats/physical_attribute_norm_stats.json
```

`water_capacity_g` 和 `capacity_ratio` 必须使用 `capacity_valid_mask`，非开口物体不参与对应 loss/metric。

## 第二阶段：诊断脚本

新增 split / 属性完整性诊断，输出：

```text
object coverage
missing physical attributes
train/val/test dry_mass_g distribution
train/val/test capacity_ratio distribution
train/val/test is_open_container distribution
train/val/test load_transition_class distribution
train/val/test shape_profile_ctrl distribution
```

当前已知风险：`split_unseen_fold1_v1` 只覆盖 OBJ001-OBJ012；而物理属性表覆盖 OBJ001-OBJ018。并且当前 train 的开口容量比范围约为 `0.397-0.826`，val/test 包含更高容量比对象，存在明显分布偏移。

## 第三阶段：物理属性头

第一版物理属性头输入仍坚持：

```text
concat(h_v, z_content)
```

不接入：

```text
tactile_low
z_med
medium_probs
medium_hidden
```

模型输出 sample-level 物理属性 logits/regression，窗口级预测仅做诊断或低权重辅助。

## 第四阶段：评价口径

同时报告：

```text
window-level physical metrics
sample-level aggregated physical metrics
object-level aggregated physical metrics
```

回归指标：

```text
MAE
RMSE
Spearman
```

分类指标：

```text
accuracy
macro F1
```

并按以下窗口集合拆分：

```text
all windows
Water only
Interface only
Air only
stable windows
stable Water only
stable Air only
stable Water + stable Air
```

## 第五阶段：控制辅助实验

物理属性头 sanity check 通过后，再比较：

```text
ref_anchor_no_attr_aux
ref_anchor_physical_attr_aux
```

此时物理属性只作为辅助监督，不输入 policy。重点观察：

```text
control_interface_mae
control_interface_hit_rate_200
control_interface_bias
force_interface_delta_mean
non_interface_delta_magnitude
```

## 暂缓事项

以下内容放到后续 ablation：

```text
oracle_metadata policy input
predicted_metadata policy input
attention pooling
metadata-conditioned delta/gate head
```

当前路线保持：

```text
reference anchor + medium-gated residual policy as main control
physical attributes as auxiliary supervision and mechanism explanation
metadata policy input as later ablation
```

当前 processed windows 中没有可观测的 `Air_to_Water` 转换，全部有效方向统计为 `Water_to_Air`，少量失败或截断样本表现为 `Water_to_Water`。因此新 split 已尽量覆盖 normal / inverse / rotate placement variants，但 medium direction 的 Air→Water 覆盖需要额外数据或重新处理原始样本。
