# 双向跨介质数据集扩充升级注意事项说明

## 1. 升级目标

当前数据集仅包含 Water-to-Air（W2A，水中→空气）方向。本次计划新增规模相近的 Air-to-Water（A2W，空气→水中）数据，将任务扩展为：

\[
Water \leftrightarrow Interface \leftrightarrow Air
\]

联合数据集用于训练方向条件化的跨介质视触融合模型，并继续保持 object-level unseen split。

本次扩充不仅是增加另一方向的数据，还会影响：

- 数据编号与元数据；
- Reference force 定义；
- 介质状态时序；
- Policy 条件输入；
- Residual 分布；
- 训练采样；
- 模型初始化；
- 评测与论文叙事。

---

## 2. 样本编号与数据组织

### 2.1 推荐编号

原 Water-to-Air 数据：

```text
S0001–S0287
```

新增 Air-to-Water 数据：

```text
S0288–S0564
```

联合后共 564 个 episode。该编号方式可行，也便于保持历史数据兼容。

### 2.2 编号不能替代方向字段

代码中不能根据 sample_id 范围推断方向，例如不得写成：

```python
direction = "W2A" if sample_id <= 287 else "A2W"
```

必须为每个 episode 显式保存方向元数据。

建议字段：

```text
sample_id
object_id
direction
source_medium
target_medium
reference_medium
split
t_grasp_stable
t_if_enter
t_if_exit
```

推荐值：

| Direction | source_medium | target_medium | reference_medium |
|---|---|---|---|
| `Water_to_Air` | Water | Air | Water |
| `Air_to_Water` | Air | Water | Air |

## 3. 对象编号与身份一致性

两个方向均使用：

```text
OBJ001–OBJ018
```

同一个 `object_id` 必须代表同一个真实物体，而不是仅表示同一类别。

必须确认：

- W2A 与 A2W 中的 OBJ001 是同一件物体；
- 对象属性元数据保持一致；
- 质量、材质、几何和开口属性等字段不发生错配；
- 同一物体在两个方向下的文件夹、标签和 sidecar 元数据可互相对应。

建议增加：

```text
physical_object_uid
```

用于避免未来重命名或复制数据导致对象身份混淆。

## 4. Object-level Unseen Split

### 4.1 保持原对象划分

继续沿用原 W2A 的 object-level unseen split。

例如原 fold1：

```yaml
train:
  OBJ001
  OBJ003
  OBJ007
  OBJ009
  OBJ011
  OBJ012

val:
  OBJ005
  OBJ010

test:
  OBJ002
  OBJ004
  OBJ006
  OBJ008
```

扩展后：

```text
train：上述 train 对象的 W2A + A2W
val：上述 val 对象的 W2A + A2W
test：上述 test 对象的 W2A + A2W
```

### 4.2 严禁跨方向泄漏

不得出现：

```text
OBJ005 的 W2A 在 train
OBJ005 的 A2W 在 val
```

同一物体的所有方向、episode、窗口、视频、触觉和缓存特征必须属于同一个 split。

建议由 object_id 自动生成 split：

```python
split = object_split_map[object_id]
```

## 5. Direction 标签设计

### 5.1 Direction 与 Medium Phase 分开

介质标签继续保持三类：

```text
Water
Interface
Air
```

新增独立方向标签：

```text
Water_to_Air
Air_to_Water
```

推荐任务分解：

\[
phase \in \{Water, Interface, Air\}
\]

\[
direction \in \{W2A, A2W\}
\]

不要把介质阶段改成 W2A_Water、A2W_Air 等六分类。

### 5.2 Direction 必须作为模型输入

`direction` 不应只用于评测和筛选，还应输入：

- Medium GRU；
- Policy Head；
- Direction-conditioned fusion；
- 可选 Direction-conditioned normalization。

推荐：

\[
e_d = Embedding(d), \quad d\in\{0,1\}
\]

## 6. 双向介质时序

Water-to-Air：

\[
Water ightarrow Interface ightarrow Air
\]

Air-to-Water：

\[
Air ightarrow Interface ightarrow Water
\]

旧 Medium GRU 若只在 W2A 上训练，可能已经学到固定单向状态转移。加入 A2W 后必须重新适配：

- GRU 隐状态转移；
- phase transition prior；
- Interface 持续步数；
- Water/Air 进入和退出模式。

推荐：

\[
h_t^m = GRU([z_t^e,e_d],h_{t-1}^m)
\]

其中 \(z_t^e\) 为触觉介质证据，\(e_d\) 为方向 embedding。

---

## 7. Reference Force 的双向统一定义

Reference force 统一定义为：

> 当前 episode 进入 Interface 之前，在源介质稳定抓取阶段测得的每指参考力。

Water-to-Air：

\[
\mathbf F_{ref}^{W2A}
=
StableForce(pre-interface\ Water\ segment)
\]

Air-to-Water：

\[
\mathbf F_{ref}^{A2W}
=
StableForce(pre-interface\ Air\ segment)
\]

统一形式：

\[
\mathbf F_{ref}
=
StableForce(source-medium\ stable\ segment)
\]

每指 residual：

\[
\Delta\mathbf F^*(t)
=
\mathbf F^*(t)-\mathbf F_{ref}
\]

Reference 必须：

- 位于 `t_if_enter` 之前；
- 基于原始触觉计算；
- 按秒定义稳定区间；
- 与 Policy 频率无关；
- 不使用 Interface 后的信息；
- 对两个方向采用相同统计规则。

推荐字段：

```text
finger_reference_forces
reference_start_time
reference_end_time
reference_medium
has_finger_reference
```

---

## 8. 不要人为翻转 A2W Residual

不能直接规定：

\[
\Delta F^{A2W}=-\Delta F^{W2A}
\]

两个方向不一定镜像对称，原因包括：

- 入水冲击；
- 水流扰动；
- 润湿迟滞；
- 表面张力非对称；
- 物体姿态变化；
- 三指载荷重新分配。

Per-finger 情况下完全可能出现：

\[
\Delta F_0<0,\quad
\Delta F_1>0,\quad
\Delta F_2<0
\]

因此必须保留真实传感器坐标和真实 signed residual。

可额外生成 `canonical_direction_residual` 用于机制分析，但不能替代原始控制标签。

## 9. 模型输入与结构调整

### 9.1 第一版推荐结构

第一版采用：

```text
共享视觉编码器
+ 共享触觉编码器
+ 共享 Medium GRU
+ direction embedding
+ 方向条件化的共享 Policy Head
```

Per-finger Policy 输入建议：

\[
h_i=
[
h_v^{global},
h_t^{global},
h_{t,i}^{local},
p_{medium},
F_{ref,i},
e_i,
e_d
]
\]

其中：

- \(h_v^{global}\)：全局视觉上下文；
- \(h_t^{global}\)：全局触觉上下文；
- \(h_{t,i}^{local}\)：第 \(i\) 根手指局部触觉；
- \(p_{medium}\)：介质信念；
- \(F_{ref,i}\)：每指参考力；
- \(e_i\)：finger embedding；
- \(e_d\)：direction embedding。

### 9.2 不建议第一版直接拆成两个独立模型

第一版不建议分别训练 W2A 和 A2W 两套模型，否则：

- 无法利用共同跨介质特征；
- 模型参数翻倍；
- 难以证明联合训练价值；
- 论文主线会退化为两个独立任务。

### 9.3 若出现方向负迁移

按以下顺序尝试：

1. Direction-conditioned FiLM；
2. Direction-conditioned LayerNorm；
3. 共享主干 + 方向专用 Policy Head；
4. 共享 direction head + 独立 magnitude head；
5. 最后再考虑完全独立模型。

## 10. 是否需要整体重新训练

加入 A2W 后，以下分布均发生变化：

- Medium 时序顺序；
- Reference 来源介质；
- Residual 正负和幅值；
- 视觉—触觉关系；
- Policy 条件空间；
- Direction 类别。

因此不能只在现有 W2A Policy checkpoint 上追加少量 epoch。

推荐 Warm-start，而不是全部随机初始化。

### Stage 1：加载 W2A 已训练权重

可加载：

- 视觉编码器；
- 触觉编码器；
- LoRA；
- Medium Evidence Encoder；
- Policy 共享部分；
- 属性头。

新增 direction embedding 随机初始化。

### Stage 2：双方向感知适配

重新训练或微调：

- Medium GRU；
- Direction-conditioned fusion；
- Tactile Evidence Encoder；
- 必要的视觉 LoRA；
- 视触对齐模块。

### Stage 3：双方向 Policy 联合训练

训练：

- per-finger residual；
- sign/direction；
- magnitude；
- gate；
- direction conditioning。

### Stage 4：低学习率联合微调

对整个可训练系统进行低学习率联合适配，减少阶段间特征不一致。

## 11. 双方向采样平衡

虽然 W2A 与 A2W episode 数量接近，但窗口数仍可能不平衡，受以下因素影响：

- episode 时长；
- Interface 持续时间；
- stable 区间长度；
- fail 样本比例；
- 每个对象有效窗口数；
- residual 正负与幅值分布。

不建议对全部窗口直接均匀随机采样。

推荐层级：

```text
object
  -> direction
      -> episode
          -> phase
              -> residual sign/magnitude
```

至少确保 batch 内：

- W2A 与 A2W 大致平衡；
- 不同 object 平衡；
- Interface 不被 stable 淹没；
- 正/负/近零 residual 均有覆盖；
- 不集中抽取同一 episode 的连续窗口。

推荐采样权重：

\[
w=
w_{object}
w_{direction}
w_{phase}
w_{residual}
\]

## 12. 归一化与统计量

必须重新计算联合训练集统计量，包括：

- tactile mean/std；
- per-finger force mean/std；
- residual mean/std；
- visual feature statistics；
- reference distribution；
- direction-specific distribution。

主训练优先使用联合训练集统一统计量，同时额外记录 W2A 和 A2W 的分方向统计。

需要检查：

- Reference 大小是否几乎可直接区分方向；
- 两方向 scale 差异是否过大；
- 统一归一化是否使某方向分辨率下降；
- 是否需要 direction-conditioned normalization。

所有统计量只能由 train split 计算。

## 15. 必须准备的训练实验

| 实验 | 训练数据 | Direction 输入 | 目的 |
|---|---|---|---|
| E1 | W2A only | 固定/无 | 保留原单方向基线 |
| E2 | A2W only | 固定/无 | 判断新方向独立难度 |
| E3 | W2A + A2W | 无 | 验证混合方向是否产生歧义 |
| E4 | W2A + A2W | 有 | 双方向主方法 |
| E5 | W2A + A2W | 有 + 方向专用 head | 检查方向负迁移 |

最关键比较：

\[
E4>E3
\]

用于证明 direction conditioning 必要。

同时检查：

\[
E4_{W2A}
\]

是否明显低于 E1。若下降明显，应排查：

- sampler 不平衡；
- direction embedding 无效；
- normalization 不合适；
- 共享 head 产生负迁移；
- Medium GRU 未充分适配。

## 17. 闭环控制评测

若模型输出用于真实执行器控制，两个方向均应完成闭环实验。

至少比较：

1. Fixed conservative force；
2. Reference-only；
3. Learned absolute force；
4. Direction-conditioned reference residual。

分别在 W2A 和 A2W 上报告：

- 成功率；
- 掉落率；
- 滑移；
- 峰值力代理；
- 三指受力不均；
- stable leakage；
- 控制延迟；
- 输出平滑度；
- 未见对象性能。

不能只在 W2A 完成在线控制、A2W 仅做离线预测，却宣称已验证双向控制。

## 18. 主要风险

### 18.1 方向捷径

模型可能根据机械臂运动方向、画面位置、水面位置、文件命名、背景差异或 episode 长度识别方向，而非理解介质动力学。

需要：

- 两方向场景设置尽量一致；
- 文件编号不进入模型；
- 做 direction-removed 消融；
- 检查视觉背景捷径。

### 18.2 Reference shortcut

空气 Reference 与水中 Reference 的数值范围可能明显不同，模型可能仅凭 Reference 大小判断方向。

需要比较：

```text
joint model without explicit direction
joint model with explicit direction
```

并分析方向信息是否仍有独立贡献。

### 18.3 双方向不是镜像

不要假设：

- A2W 一定更容易；
- Residual 必然反号；
- Interface 时长一致；
- 两方向最佳 gate 相同；
- 相同对象反向轨迹完全对称。

### 18.4 原 W2A 性能退化

联合训练后必须单独比较原 W2A 指标。

若性能明显下降，应考虑：

- direction-specific head；
- direction-conditioned normalization；
- 分方向 loss weight；
- 分阶段训练；
- 更严格的方向平衡采样。

## 19. 推荐实施顺序

### Phase 1：新增数据整理

- 分配 S0288–S0564；
- 核对 OBJ001–OBJ018；
- 添加 direction/source/target/reference medium；
- 完成事件标注；
- 完成 A2W per-finger 曲线检查。

### Phase 2：双向数据管线

- 更新 samples/windows schema；
- 按 source medium 构建 Reference；
- 构建双方向 Medium 标签；
- 保持 object-level unseen split；
- 重新统计训练集归一化参数。

### Phase 3：感知模型适配

- 加入 direction embedding；
- 双方向重新适配 Medium GRU；
- 检查两个方向的 Interface F1；
- 验证 direction conditioning。

### Phase 4：Policy 联合训练

- 加载 W2A checkpoint warm-start；
- 双方向联合训练 per-finger residual；
- 继续 sign/magnitude 诊断；
- 检查方向负迁移。

### Phase 5：消融与评测

- 完成 E1–E5；
- 分别报告 W2A/A2W/Macro；
- 进行 object-level unseen 统计；
- 分析 fail case。

### Phase 6：真实闭环

- 两方向均完成在线控制；
- 比较 fixed/reference-only/absolute/residual；
- 报告实时性和任务收益。

## 21. 一句话实施原则

> 本次升级不是将 A2W 样本简单追加到 W2A 数据末尾，而是把任务正式重定义为方向条件化的双向跨介质参考锚定残差控制问题；方向必须进入模型，Reference 必须始终来自源介质稳定阶段，同一物体的两个方向必须严格保持在同一个 unseen split 中。
