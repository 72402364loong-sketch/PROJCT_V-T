V1 定位

我建议把 V1 定成一句话：

在不改当前 reference_anchor + delta target 语义的前提下，给 residual 分支增加一份“界面前 stable context 摘要”，看它能不能把 delta 学得更充分。

这版最小方案里，我会主动放弃两件事：

不做新的 target 体系
不让 context 影响 gate/medium 路径
这样 V1 才够干净。

数据方案

先做一个很轻的 manifest，不要重复落 target，只落“索引和掩码”。

建议新增 build_interface_context_manifest.py，输出 data/processed/interface_context/fold1_manifest.csv，每行一个 sample，字段只要这些：

sample_id
object_id
subset
context_window_indices_json
stable_context_window_indices_json
interface_window_indices_json
context_count
stable_context_count
interface_count
has_context
used_fallback_context
context_window_mask 的定义我建议直接定死为：

主规则：window_center ∈ [t_grasp_stable, t_if_enter) 且 is_stable_mask == 1
fallback：如果主规则为空，用 [t_grasp_stable, t_if_enter) 里最后 1-2 个非 Interface 窗口
训练首版可以更干净一点：直接过滤掉没有 stable context 的样本
这一步是站得住的。我刚统计过 fold1，可用样本量够：

train 94
val 31
test 62
而且大多数样本都有 stable context。

Dataset 方案

这里我反而建议你放宽“绝不动当前 dataset”的约束。

V1 最省事也最不容易漂 target 的做法，不是新造一整套 dataset，而是对现有 dataset.py 做只增字段扩展：

加载 manifest

其余现有字段全部不动，尤其：

这样 loss 和诊断都能原样复用。

模型方案

V1 不需要新 backbone，也不需要 cross-attention。

最小可行做法是在 system.py 里新增一条 context 汇总支路：

先构造每个窗口的局部 context feature：

context_source_t = concat(h_v_t, z_content_t, z_med_window_t)
我推荐用这个，而不是 task_context 或 medium_sequence_features，原因是：

它确实来自当前窗口的本地视觉/触觉状态
不会把对象头或整段 GRU 的全局信息提前混进去
更符合“界面前 stable 状态摘要”的语义
然后按 context_window_mask 做 masked mean pooling：

context_embedding = mean(context_source_t over context_window_mask)
再把这份 context_embedding 只注入 residual path，不碰 gate：

residual_hidden = gelu(task_proj(current) + state_proj(current))
residual_hidden = residual_hidden + alpha * context_proj(context_embedding)
delta_pred = residual_head(FiLM(residual_hidden, p_medium))
force_pred = reference_force + gate * delta_pred
这里我强烈建议：

context_proj: Linear(896 -> 256)
alpha 是可学习标量，初始化为 0
或者 context_proj 最后一层零初始化
这样从旧 checkpoint 初始化时，模型起点几乎等于 baseline，不会因为新支路一接上就把已有解打坏。

当前维度是合适的：

h_v = 512
z_content = 256
z_med_window = 128
所以 context_source_dim = 896
policy hidden 本来就是 256
这些都能直接接到现有 PolicyHead 的 residual hidden 空间里。

训练方案

V1 第一条正式 run，我建议这样定：

从 balanced best 初始化，不从 stronger 起跑
新支路零初始化
base_source 继续 reference_force
loss 完全沿用当前 balanced
冻结策略基本沿用当前 stage36b reference_anchor 路线
首跑只新增训练：
policy_head.context_proj
policy_head.context_alpha
policy_head.residual_hidden_layer
policy_head.residual_output_layer
不建议一上来把整条 residual path 全放开，也不建议去动 medium_head。

建议新 stage 叫：

configs/stages/stage36b_policy_fold1_reference_anchor_delta_context_v1.yaml

评估方案

主评估口径不变，继续用现有 diagnose_policy_control.py。

首批只盯这几个：

control_interface_mae
control_interface_hit_rate_200
control_interface_bias
delta_minus_target
stable_expert.force_interface_delta
stable_expert.gated_residual
我建议额外再加两组分桶分析，不改主口径，只补视角：

按 has_interface_context / stable_context_count
按 |delta_target| 大小
因为 V1 很可能不是全局都涨，而是“有足够 stable context 的难界面样本”先涨。

最小改动清单

我建议实际只做这 5 处：

新增 build_interface_context_manifest.py
小改 dataset.py 和 collate，只增 context_window_mask
小改 system.py，构造 context_embedding
小改 modules.py，给 PolicyHead 增加 context_proj/context_alpha
新增一条 V1 stage yaml
loss、trainer、diagnose 都先不动。
