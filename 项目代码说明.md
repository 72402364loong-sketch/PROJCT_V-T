# 项目代码说明

## 1. 项目是做什么的

这是一个跨介质视触抓取学习项目，核心目标是把视频信息和触觉信息联合起来，完成以下几类任务：

- 识别当前窗口属于 `Water / Interface / Air` 哪个介质阶段
- 预测对象属性，如 `fragility / geometry / surface`
- 学习视觉表征与触觉表征之间的对齐关系
- 根据当前视觉、触觉和介质信念输出抓取力策略

整个工程按“数据预处理 -> 切分 -> Dataset -> 模型 -> 损失 -> 训练/评估”组织。

## 2. 顶层目录对应什么功能

### `scripts/`

命令行入口脚本目录。

- `scripts/preprocess.py`
  - 预处理入口。
  - 调用 `cmg.data.write_preprocessed_outputs(...)` 生成 `samples.csv`、`windows.csv` 和统计文件。
- `scripts/train.py`
  - 训练入口。
  - 负责读取基础配置与阶段配置，构建数据集、DataLoader、模型和 `Trainer`，并执行训练。
  - `--resume` 只用于同一 `run/stage` 断点续训；`--init-from` 用于跨阶段权重初始化，并允许在 Stage1 -> Stage2 / Stage4 时注入新的视觉 LoRA 参数。
- `scripts/evaluate.py`
  - 评估入口。
  - 按指定阶段配置和 checkpoint 加载模型，在 `train/val/test` 子集上跑一轮评估并输出 JSON 指标。
  - 评估结果写入独立的 `evals/` 目录，不污染训练 run 目录。
- `scripts/start_stage1_formal.ps1`
  - Windows 下的正式 Stage1 后台启动脚本。
  - 通过独立的 PowerShell host 进程承载训练，关闭 Codex 或当前 PowerShell 后训练仍可继续；默认优先续上 `stage1_latest.pt`。
- `scripts/watch_stage1_formal.ps1`
  - 查看正式 Stage1 的后台任务状态、最近 metrics 和实时 stderr 日志。
- `scripts/stop_stage1_formal.ps1`
  - 停止正式 Stage1 的独立训练进程。

### `src/cmg/`

项目核心源码目录，`cmg` 是主 Python 包。

- `src/cmg/config.py`
  - 提供 `load_yaml()` 和 `deep_update()`。
  - 用于读取 YAML 配置，并把 stage 配置覆盖到默认配置上。
- `src/cmg/constants.py`
  - 定义类别到索引的映射，如介质阶段、易碎性、几何类别、表面类别。
  - 还定义了主任务对象池和机制预训练对象池，并通过 `infer_object_pool()` 判断对象属于哪个池。
- `src/cmg/losses.py`
  - 定义训练时使用的损失函数。
  - 包含：
    - `info_nce_loss()`：视觉和触觉对齐的对比损失
    - `supcon_cross_medium_loss()`：跨介质监督对比损失
    - `compute_losses()`：把 `med / attr / clip / inv / pol` 各项损失汇总成总损失
- `src/cmg/training.py`
  - 训练器和评估逻辑所在模块。
  - 包含优化器、学习率调度器、冻结模块、checkpoint 保存/恢复、指标计算、TensorBoard 记录等。

### `src/cmg/data/`

数据处理与数据加载模块。

- `src/cmg/data/annotations.py`
  - 负责把原始标注 CSV 标准化。
  - `normalize_object_attributes()` 统一对象属性表字段和类别命名。
  - `normalize_sample_events()` 统一样本事件表字段、时间戳、trial 编号、同步偏移等。
- `src/cmg/data/preprocess.py`
  - 负责从原始标注和原始模态数据生成训练所需表格。
  - `build_samples_table()`：把对象标注和样本事件合并为样本主表。
  - `build_windows_table()`：按时间窗口切分视频/触觉数据，生成窗口级元数据。
  - `write_preprocessed_outputs()`：把结果写入 `data/processed/`。
- `src/cmg/data/splits.py`
  - 负责解析 split 配置，并根据 `train / val / test` 返回对应 `sample_id`。
  - 支持两种 split：
    - `object_level_unseen`
    - `sample_level_random`
- `src/cmg/data/video.py`
  - 负责视频侧处理。
  - 包括：
    - 读取视频元数据
    - 根据时间窗口生成帧索引
    - 从候选帧中均匀采样固定数量帧
    - 解码视频帧、裁剪 ROI、缩放到固定尺寸
- `src/cmg/data/tactile.py`
  - 负责触觉侧处理。
  - 包括：
    - 读取原始 36 维触觉序列
    - EMA 滤波
    - AC/DC 高频低频分解
    - 窗口重采样
    - 标准化
    - 根据稳定抓取样本估计专家力曲线
- `src/cmg/data/windowing.py`
  - 负责窗口标签规则。
  - 根据窗口与 `t_if_enter / t_if_exit` 的关系计算：
    - `phase_label`
    - `is_stable_mask`
    - `stable_phase`
    - `tail_type`
- `src/cmg/data/dataset.py`
  - 项目最核心的数据集实现。
  - `CrossMediumSequenceDataset` 会：
    - 读取预处理后的 `samples.csv` 与 `windows.csv`
    - 根据 split 过滤样本
    - 过滤 trial 结果、低覆盖率窗口和尾部窗口
    - 加载视频窗口和触觉窗口
    - 计算标签与专家力监督
    - 返回“一个样本由多个窗口组成”的序列样本
  - `sequence_collate_fn()` 负责把不同长度的样本序列补齐后拼成 batch。
- `src/cmg/data/sampler.py`
  - `ObjectAwareBatchSampler`
  - 训练时让同一对象的样本按设定数量进入 batch，提升 object-aware 训练稳定性。
- `src/cmg/data/__init__.py`
  - 对外导出数据模块最常用的类与函数。

### `src/cmg/models/`

模型结构模块。

- `src/cmg/models/modules.py`
  - 放所有基础网络组件。
  - 关键内容包括：
    - `VisualEncoder`
      - 视觉编码器
      - 支持 `open_clip ViT-B-16`、`torchvision vit_b_16`、`resnet18` 和一个兜底的 `SmallCNNBackbone`
      - 支持 q/v 语义 LoRA，并可按 stage 控制是否注入或冻结视觉适配器
      - 当前正式链路采用：Stage1 不注入视觉 LoRA，Stage2 / Stage4 开启视觉 LoRA，Stage3 保留 LoRA 结构但跟随 `visual_encoder` 一起冻结
      - 用时序注意力把多帧聚合成窗口级视觉特征
    - `TactileContentEncoder`
      - 对高频触觉序列编码
      - 用传感器 MLP + Transformer 编码内容表征
      - 输出内容特征和对齐用投影特征
    - `TactileEvidenceEncoder`
      - 对低频触觉序列编码
      - 用 1D CNN + 注意力池化提取介质证据特征
    - `MediumBeliefHead`
      - 用 GRU 在窗口序列上递推介质信念
      - 输出每个窗口的 `Water / Interface / Air` logits 和概率
    - `MultiAttributeHead`
      - 根据视觉特征和触觉内容特征预测对象属性
      - 同时构造对象上下文特征 `g_obj_context`
    - `PolicyHead`
      - 根据任务上下文和介质概率做 FiLM 条件化
      - 输出力预测 `force_pred`
- `src/cmg/models/system.py`
  - `CrossMediumSystem`
  - 负责把各个子模块拼成完整系统。
  - 前向过程大致是：
    1. 视频窗口 -> 视觉编码器
    2. 高频触觉 -> 内容编码器
    3. 低频触觉 -> 证据编码器
    4. 证据序列 -> 介质信念头
    5. 视觉 + 内容 -> 属性头
    6. 视觉 + 内容 + 对象上下文 + 介质概率 -> 策略头
    7. 返回训练和评估需要的全部中间结果与预测结果
- `src/cmg/models/__init__.py`
  - 对外导出 `CrossMediumSystem`。

### `configs/`

配置目录，控制数据处理、模型结构、训练超参数和阶段化训练策略。

- `configs/data/default.yaml`
  - 数据相关默认配置。
  - 包括视频 FPS、触觉采样间隔、窗口大小、窗口步长、ROI、采样帧数、触觉标准化等。
- `configs/model/default.yaml`
  - 模型相关默认配置。
  - 包括视觉骨干、LoRA 参数、触觉编码器维度、介质头、属性头、策略头和损失温度参数等。
- `configs/train/base.yaml`
  - 训练相关默认配置。
  - 包括 epoch、batch size、学习率、warmup、AMP、梯度裁剪、early stopping、采样器和损失权重等。
- `configs/stages/*.yaml`
  - 阶段训练配置。
  - 会覆盖默认配置，用于定义每个训练阶段的：
    - 数据 split
    - 冻结哪些模块
    - 是否在该 stage 注入或冻结视觉 LoRA
    - 启用哪些损失
    - 采用什么最优模型选择指标

### `data/`

数据目录。

- `data/annotations/`
  - 原始标注表。
  - `object_attributes.csv` 是对象级属性标注。
  - `sample_events.csv` 是样本级时间事件标注。
- `data/raw/videos/`

- `data/raw/tactile/`

- `data/processed/`

  - 关键输出：
    - `samples.csv`

    - `stats/dataset_summary.json`
    - `cache/` 下的触觉标准化统计缓存
- `data/splits/`

## 3. 主流程是怎么串起来的

### 3.1 预处理流程

入口：`scripts/preprocess.py`

调用链：

1. `write_preprocessed_outputs()`
2. `build_samples_table()`
3. `normalize_object_attributes()` + `normalize_sample_events()`
4. `build_windows_table()`
5. `label_window()`、`determine_tail_type()`、`get_video_metadata()`、`load_tactile_array()`
6. 输出到 `data/processed/`

这一步的作用是把“对象标注 + 样本事件 + 原始视频/触觉长度信息”整理成后续训练能直接使用的结构化表格。

### 3.2 训练流程

入口：`scripts/train.py`

调用链：

1. `load_configs()` 读取默认配置和 stage 配置并合并
2. `make_dataset()` 构建训练集和验证集
3. `make_train_loader()` 构建 DataLoader，必要时启用 `ObjectAwareBatchSampler`
4. `CrossMediumSystem(model_config)` 构建模型
5. `Trainer(...)` 构建训练器
6. `trainer.fit()` 执行训练、验证、checkpoint 保存和指标记录

这里需要特别注意：

- `--resume` 只恢复同一 stage 的训练状态
- `--init-from` 只加载模型权重，不恢复 optimizer / scheduler / scaler / epoch
- 当目标 stage 首次打开视觉 LoRA 时，`--init-from` 会自动接受新增 `lora_*` 参数缺失，并重映射 OpenCLIP attention 权重名

### 3.3 单个 batch 的数据流

一个样本对应一个抓取序列，序列里包含多个时间窗口。

在 `CrossMediumSequenceDataset` 中：

1. 从 `windows.csv` 找到该样本所有窗口
2. 每个窗口加载固定数量视频帧
3. 每个窗口切出对应触觉片段
4. 触觉拆成高频分量和低频分量
5. 为每个窗口构造介质标签、稳定掩码和专家力标签
6. `sequence_collate_fn()` 把变长序列补齐成 batch

在 `CrossMediumSystem.forward()` 中：

1. 视频窗口得到视觉特征 `h_v` 和对齐特征 `z_v`
2. 高频触觉得到内容特征 `z_content` 和对齐特征 `z_t`
3. 低频触觉得到介质证据特征 `z_med_window`
4. 窗口证据序列进入 GRU，得到介质分类结果
5. 视觉特征和内容特征进入属性头，得到属性预测
6. 属性上下文和介质概率一起进入策略头，得到力预测

### 3.4 损失与指标

`compute_losses()` 会把模型输出和 batch 标签对应起来，计算：

- `med`
  - 介质分类交叉熵
- `attr`
  - 属性分类损失之和
- `clip`
  - 视觉/触觉对齐损失
- `inv`
  - 跨介质监督对比损失
- `pol`
  - 力预测均方误差
- `total`
  - 按阶段权重加权后的总损失

`Trainer.run_epoch()` 额外会统计：

- 介质分类 accuracy、macro F1、各类别 F1
- 属性分类 accuracy、macro F1、联合属性准确率
- 力回归的 `overall / stable / interface` MAE 与 MSE
- 混淆矩阵

## 4. 分阶段训练配置分别做什么

### `stage0_mechanism.yaml`

- 用 `probe_mechanism_v1.yaml` 机制对象池做预训练
- 启用 `clip / inv / med / attr`
- 不训练策略力预测
- 关注 `medium_macro_f1`

### `stage1_perception_fold1.yaml`

- 面向主任务 fold1 的感知训练阶段
- 训练介质识别和属性识别
- 不启用对齐损失和策略损失
- 视觉 backbone 冻结，且当前正式配置不注入视觉 LoRA
- 关注 `joint_score`

### `stage2_alignment_fold1.yaml`

- 在 stage1 基础上加入视觉-触觉对齐与跨介质一致性约束
- 启用 `clip + inv + med + attr`
- 重新打开视觉 LoRA，让视觉适配器在对齐目标下开始训练
- 仍不训练策略头

### `stage2_joint_fold1.yaml`

- 配置上和 `stage2_alignment_fold1.yaml` 基本一致
- 同样会重新打开视觉 LoRA
- 可以视为 stage2 的联合训练版本命名

### `stage3_policy_fold1.yaml`

- 冻结大部分感知和属性模块
- 保留视觉 LoRA 结构，但跟随 `visual_encoder` 一起冻结
- 只重点训练策略相关部分
- 仅启用 `pol`
- 关注 `interface_mae`

### `stage4_joint_fold1.yaml`

- 解冻全模型做联合微调
- 同时优化感知、对齐和策略
- 视觉 LoRA 在这一阶段重新参与联合微调
- 关注 `interface_mae`

## 7. 一句话总结

这个项目的代码结构比较清晰，可以把它理解为：

- `data/` 负责把原始标注和原始模态整理成“可训练窗口序列”
- `models/` 负责把视觉、触觉、介质信念、对象属性和策略预测拼成一个完整系统
- `losses.py` 负责定义训练目标
- `training.py` 负责把训练、验证、指标和 checkpoint 串起来
- `scripts/` 和 `configs/` 负责把整条链路真正跑起来
