# 跨介质视触抓取项目论文基座总结

更新日期：2026-06-27  
项目根目录：`D:\PROJCT_V-T`  
口径说明：本文以当前工作区最新代码、配置、运行结果与已有项目文档为准，包含尚未提交到 Git 的三指执行器对齐力空间改造。

## 1. 论文定位

本项目研究“跨介质视触抓取”问题：机械臂在水下完成抓取后，夹持目标上提并穿越水-气界面，最终在空气中保持稳定抓取。相较常规抓取，该任务的核心困难不是单次静态抓取姿态，而是介质状态变化引起的接触力需求变化，以及视觉、触觉、对象属性、样本级执行状态之间的耦合。

当前最适合的论文主线不是“端到端绝对力回归”，而是：

```text
reference-anchored residual force adaptation for cross-medium grasping
```

即使用当前 episode 稳定阶段测得的 `reference_force` 作为控制锚点，让模型学习跨越界面时相对锚点的残差修正：

```text
force_pred = reference_force + gate * delta
```

最新实现进一步把旧的 scalar grasp force regression 扩展到三指执行器对齐的 per-finger force distribution learning：

```text
F_pred_i(t) = F_ref_i(e) + g(t) * DeltaF_i(t),  i in {0, 1, 2}
```

因此，论文可被组织为：面向水-气跨介质抓取的视触融合感知、阶段信念估计与参考锚定残差力自适应方法，其中最新控制输出空间与三指夹爪的独立执行接口对齐。

## 2. 可用论文题目

1. 面向水-气跨介质抓取的视触融合参考锚定残差力控制
2. Cross-Medium Visual-Tactile Grasping via Reference-Anchored Residual Force Adaptation
3. Actuator-Aligned Per-Finger Residual Force Learning for Cross-Medium Robotic Grasping

## 3. 摘要草稿

水下机器人在抓取目标并穿越水-气界面时，接触状态、浮力、流体阻力和对象负载会共同改变合理抓取力，使常规静态抓取策略难以直接适用。本文构建了一个跨介质视触抓取学习框架，将视频窗口、触觉时序、介质阶段信念和对象属性表征联合建模，并将策略学习从直接绝对力回归重构为参考锚定残差控制。模型使用 OpenCLIP ViT-B/16 与 q/v LoRA 提取视觉表征，采用高低频触觉分支分别服务对象/接触内容编码与介质证据建模，通过 GRU 递推估计 Water、Interface、Air 三阶段信念。控制侧以 episode 内稳定抓取段测得的 `reference_force` 为锚点，仅学习界面阶段残差修正 `delta`，从而绕开对象级绝对基准力泛化不稳定的问题。最新版本进一步将力预测从标量合力推广到三指执行器对齐的每指力向量，使输出可直接对应三指夹爪控制接口。实验表明，纯 learned absolute base 在 fold1 unseen-object 验证中几乎失败，而 reference-anchor residual 将 scalar control interface MAE 从约 519 降至约 68；三指 per-finger 版本在 per-finger interface control MAE 上达到约 8.78，并暴露出 residual 幅度仍偏保守的下一步优化方向。

### 4.2 主 split

经典主实验使用 object-level unseen fold1：

```yaml
val:   OBJ005, OBJ010
test:  OBJ002, OBJ004, OBJ006, OBJ008
```

```yaml
train: OBJ001, OBJ003, OBJ007, OBJ009, OBJ010, OBJ011, OBJ012, OBJ013, OBJ017, OBJ018
val:   OBJ005, OBJ006, OBJ014, OBJ016

```

该 split 覆盖 OBJ001-OBJ018，平衡了开口/非开口对象、干重、容量比和 Interface 窗口支持。但当前 processed windows 中有效转移方向只有 Water_to_Air，缺少 Air_to_Water 覆盖，这是论文边界条件。

### 4.3 标签体系

项目当前同时维护四类标签：

| 标签类型 | 内容 | 论文定位 |
|---|---|---|
| 介质阶段 | Water / Interface / Air | 主感知任务 |
| 语义属性 | fragility / geometry / surface 及 coarse_v2 | 辅助对象理解 |
| 物理属性 | dry_mass_g / capacity_ratio / is_open_container | 机制解释和辅助监督 |
| 控制标签 | reference_force / delta_force_target / per-finger force | 主策略任务 |

语义属性已从 legacy taxonomy 收缩到 coarse_v2：

```text
fragility_v2: fragile / non_fragile
shape_profile_v2: open_vessel / slender_or_narrow
surface_texture_v2: smooth / textured_or_matte
```

当前最稳妥的语义属性主指标是 `shape_profile_v2`，不建议将 fragility 或 surface 作为论文核心指标。物理属性头已实现，但目前应定位为 auxiliary supervision 与 mechanism probing，而不是直接输入 policy 的主方法。

## 5. 方法框架

### 5.1 总体结构

模型主干为 `CrossMediumSystem`，由以下模块构成：

```text
video windows
  -> VisualEncoder
  -> h_v, z_v

tactile_high
  -> TactileContentEncoder
  -> z_content, z_t

tactile_low
  -> TactileEvidenceEncoder
  -> z_med_window
  -> MediumBeliefHead(GRU)
  -> p_medium

concat(h_v, z_content)
  -> MultiAttributeHead / PhysicalAttributeHead
  -> object context

task context + medium/state context + reference context
  -> PolicyHead
  -> force / per-finger force prediction
```

### 5.2 视觉分支

视觉主线采用实时 OpenCLIP ViT-B/16 + q/v LoRA：

- backbone：`open_clip_vit_b_16`
- LoRA 注入位置：OpenCLIP transformer attention 的 Q/V 投影
- 默认 rank：8；增强实验 rank：16
- 多帧窗口通过 `VisualTemporalAttentionPooling` 聚合
- `max_windows_per_encode` 用于控制显存峰值

Stage1 可使用 frozen visual cache 做低成本感知训练；Stage2/Stage4 打开实时 ViT + LoRA，用 `L_clip` 和 `L_inv` 做视触对齐。

### 5.3 触觉高低频分支

触觉输入原始为 36 维，即 3 根手指、每指 4 个 taxel、每个 taxel 3 轴。当前触觉被 AC/DC 分解：

- `tactile_high`：高频内容分支，输入 `TactileContentEncoder`，服务对象属性、视触对齐、策略上下文。
- `tactile_low`：低频证据分支，输入 `TactileEvidenceEncoder` 和 `MediumBeliefHead`，服务介质阶段信念与 gate。

高低频解耦已有明确代码实现，但还缺少 high-only、low-only、shared encoder、visual-only、tactile-only 等系统消融。因此论文里应将其作为模型结构动机和辅助设计，而不是最强理论贡献。

### 5.4 介质信念与 gate

`MediumBeliefHead` 使用 GRU 对窗口序列递推，输出每个窗口的 `p_medium = [p_water, p_interface, p_air]`。策略中的默认 gate 使用 `p_interface`，也测试过 soft target、hard phase oracle 和 `PhaseAwareGateStabilizer`。

当前结论是：更干净的 oracle gate 能压低 stable leakage，但不一定提升 interface 控制指标。medium_prob gate 的提前/滞后激活可能对界面控制有正贡献。

### 5.5 策略头：从标量到三指执行器空间

旧标量主线为：

```text
force_pred = reference_force + interface_gate * force_interface_delta
```

最新三指主线为：

```text
F_pred(t) = F_ref(e) + g(t) * DeltaF(t),    F in R^3
```

每指标签由同一手指 4 个 z 轴传感器构造：

```text
F_i(t) = mean_j abs(z_i,j(t) - baseline_i,j)
```

其中 baseline 默认来自接触前窗口的 median。该定义避免了旧 signed-sum 在三指圆周 120 度布置下因局部坐标轴不一致产生的虚假抵消。

新增 per-finger 字段包括：

```text
finger_expert_forces
finger_control_force_targets
finger_reference_forces
finger_delta_force_targets
has_finger_expert
has_finger_control_target
has_finger_reference
```

新增 policy head：

```yaml
policy:
  head_type: state_residual_per_finger
  finger_count: 3
  finger_embedding_dim: 16
```

其核心是共享 task/state context，并通过 finger embedding 输出 `[F0, F1, F2]`。

## 7. 当前实验结果摘要

### 7.1 感知与属性

| run | medium macro F1 | Interface F1 | 属性主指标 |
|---|---:|---:|---:|
| `stage1_perception_fold1` | 0.8446 | 0.6675 | legacy attr 弱 |
| `stage1_perception_fold1_attr_v2` | 0.8418 | 0.6579 | V2 属性更稳 |
| `stage2_alignment_fold1_lowmem_inv_attr_v2_sample_pool` | 0.8368 | 0.6465 | `attr_primary_macro_f1=0.5735` |
| `stage2_alignment_fold1_lowmem_inv_attr_v2_shape_only` | 0.8461 | 0.6650 | shape-only 主指标稳定 |
| `stage2_alignment_fold1_lowmem_inv_attr_v2_shape_only_lora16_tactile3` | 0.8400 | 0.6496 | shape 略增，interface 略降 |

关键判断：

- medium 感知整体可用，`medium_macro_f1` 常在 0.84-0.88。
- Interface 仍是最难阶段，`medium_f1_interface` 常在 0.65-0.75。
- 属性任务应采用 sample/object 聚合评价，不应只看 all-window 指标。
- `shape_profile_v2` 是最适合写入论文主结果的对象属性口径。

### 7.2 scalar reference-anchor 控制主证据

| run | control interface MAE | hit rate @200 | 结论 |
|---|---:|---:|---|
| `stage36b_policy_fold1_base_norm` | 519.32 | 0.0577 | pure learned absolute base 基本失败 |
| `stage36b_policy_fold1_reference_anchor_delta_balanced` | 68.33 | 0.9399 | reference anchor 带来数量级改进 |
| `stage36d_reference_anchor_medium_gate_stabilized_delta` | 68.32 | 0.9351 | stabilizer 略改善，但非质变 |
| `stage36d_b_physical_balanced_reference_anchor_medium_gate_stabilized_relaxed_delta` | 70.59 | 0.9369 | physical-balanced split 上仍保持主线有效 |

这是目前最强的论文证据链：绝对 base 泛化失败，而 measured `reference_force` 直锚能显著降低界面控制误差。

### 7.3 large-delta 诊断

scalar residual 诊断显示，大幅正/负界面修正不只是 gate 问题，更是 residual 输出尺度、方向和幅度监督问题。C2 direction + magnitude supervised 诊断版取得当前 large-delta 最好综合表现：

| run | large-delta balanced score | large-delta interface MAE | 结论 |
|---|---:|---:|---|
| B0 scaled residual | 约 163.08 | 约 141.04 | 输出尺度是关键瓶颈之一 |
| C sign-specific | 约 164.04 | 约 144.41 | 方向 head 有容量但路由不稳 |
| C1 direction supervised | 约 165.66 | 约 143.43 | direction gate 可学但幅度仍不足 |
| C2 direction + magnitude supervised | 154.18 | 134.74 | 当前 large-delta 诊断最好 |

C2 的意义是证明幅度监督有效，但仍未完全解决样本/对象条件下的正负方向分离。

### 7.4 三指 per-finger 控制

Stage38 是当前最新控制输出口径。注意：Stage38 中旧 scalar `control_*` 指标仅为兼容日志，不应作为论文主指标；论文应报告 `finger_*` 指标。

| run | epoch | finger control interface MAE | finger delta interface MAE | finger0 / finger1 / finger2 interface MAE |
|---|---:|---:|---:|---|
| `stage38_policy_fold1_per_finger_reference_delta` | 1 | 8.94 | 8.94 | 7.73 / 8.58 / 10.50 |
| `stage38b_policy_fold1_per_finger_reference_delta_scale20_freeze` | 12 | 8.78 | 8.68 | 7.63 / 8.59 / 10.10 |

当前判断：

- per-finger 力空间已完整接入 Dataset、System、PolicyHead、Loss、Training metrics。
- Stage38b 通过 `delta_normalization_scale: 20.0` 和冻结 `attribute_head.object_projection` 取得小幅改进。
- 预测仍偏接近 reference，说明每指 residual 学习信号仍偏弱，下一步应继续强化 per-finger residual 幅度、direction/magnitude 或 local tactile branch。

### 7.5 物理属性头

物理属性头已接入数据管线和模型，输出：

```text

sample_capacity_ratio_normalized_pred
```

|---|---:|---:|---:|---:|
| `stage1_perception_physical_balanced` | 0.8705 | 0.7228 | 90.71 g | 0.5688 |

## 8. 可提炼的论文贡献

### 贡献 1：跨介质视触抓取任务与数据处理闭环

项目建立了从原始视频、触觉和事件标注到窗口级训练样本的完整离线闭环，支持 object-level unseen split、阶段标签、稳定参考段、界面修正段和 sidecar cache。

### 贡献 2：参考锚定残差控制范式

与直接预测绝对抓取力相比，reference-anchor residual 将问题重构为：

```text
当前 episode 稳定执行状态提供锚点；
模型只学习跨介质界面造成的相对修正。
```

这条主线有最强实验支撑，是论文核心贡献。

### 贡献 3：视触融合介质信念与残差 gate

视觉提供对象和场景上下文，触觉高低频分支分别提供接触内容和介质证据，GRU medium head 估计阶段信念，`p_interface` 作为 residual gate。oracle gate 诊断说明 gate 不是唯一瓶颈，medium_prob 的软激活具有实际价值。

### 贡献 4：三指执行器对齐的 per-finger 输出空间

最新实现将控制目标从虚拟合力推广到三维每指力向量，并给出基于 z 轴传感器 baseline-corrected absolute mean 的 per-finger force proxy。这使方法从“可评估”进一步靠近“可执行”。

### 贡献 5：物理属性辅助监督与机制解释支线

项目引入 dry mass、capacity ratio、is-open-container 等对象级物理属性，用于解释跨介质负载变化和辅助表征学习。当前结果适合作为机制分析或未来工作，不应压过 reference-anchor 主线。

## 11. 当前局限

1. 数据规模仍小，主 fold1 val 只有 2 个对象，结果对 object 分布敏感。
2. processed 数据当前只有 Water_to_Air，有效 Air_to_Water 转移缺失。
3. high/low tactile 解耦尚缺系统消融，不能单独作为强理论贡献。
4. legacy semantic attributes 与 object identity 绑定较强，fragility/surface 不宜作为主指标。
5. physical attributes 仍是早期 sanity，capacity_ratio 和 is_open 结果不够稳。
6. scalar reference-anchor 已有强结果，但 per-finger residual 仍偏保守，Stage38b 只是小幅推进。
7. 三指 per-finger 版本尚未加入 local per-finger tactile branch，当前仍使用 global context + finger embedding。

## 12. 下一步最小实验矩阵

### 12.1 per-finger 主线

优先级最高：

1. 对 Stage38b 做 per-sample/per-finger 曲线可视化，确认 8.78 MAE 是否主要来自贴 reference。
2. 增加 per-finger residual output scale 或 sign/magnitude 监督。
3. 增加 local per-finger tactile branch，对比 `global_context + finger_embedding`。
4. 引入 per-finger force-balance regularization，观察三指误差差异是否下降。

### 12.2 scalar 论文主证据补齐

建议保留：

1. learned absolute base baseline。
2. reference-anchor balanced 主结果。
3. medium gate stabilizer 附加结果。
4. oracle gate 结果作为诊断，而不是主方法。

### 12.3 物理属性与机制解释

建议顺序：

1. 继续完善 physical-balanced split 上 Stage1/Stage2 属性头。
2. 报告 sample/object 级物理属性指标。
3. 在控制阶段只做 auxiliary physical loss，不直接输入 policy。
4. 最后再做 oracle_metadata / predicted_metadata 消融。

### 12.4 必要消融

为支撑论文完整性，最小消融建议：

| 消融 | 目的 |
|---|---|
| no reference / learned base / reference anchor | 证明主贡献 |
| medium_prob gate / hard oracle gate / soft oracle gate | 说明 gate 不是唯一瓶颈 |
| scalar force / per-finger force | 证明执行器对齐输出空间价值 |
| cache visual / realtime ViT + LoRA | 说明视觉适配路线 |

## 13. 一句话结论

截至 2026-06-27，本项目已经从“端到端绝对力预测”收敛为“episode-level measured reference_force 锚点 + medium-gated residual delta 修正”的跨介质视触抓取控制框架；scalar 控制主线已有强证据，最新工程正在把该框架推进到三指执行器对齐的 per-finger 力空间。论文应以 reference-anchored residual control 为核心贡献，以实时 ViT+LoRA、触觉高低频分支、物理属性头和 per-finger 输出作为支撑与扩展。
